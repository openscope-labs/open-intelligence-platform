1. RINGKASAN EKSEKUTIF
--------------------------------------------------------------------------------

Project Tracking & Intelligence Platform adalah platform perangkat lunak
berbasis web yang dirancang untuk mendukung operasi Search and Rescue (SAR),
khususnya pelacakan dan pencarian orang hilang di area pegunungan. Platform
ini mengintegrasikan teknologi pelacakan geolokasi real-time, peta interaktif
multi-layer, analisis berbasis kecerdasan buatan, dan komunikasi terpadu
dalam satu antarmuka.

1.1 Tujuan Sistem
  - Menyediakan Common Operating Picture (COP) real-time untuk koordinasi
    operasi SAR.
  - Melacak posisi seluruh anggota tim lapangan secara real-time pada peta
    interaktif multi-layer.
  - Mengintegrasikan data dari berbagai sumber (GPS, CCTV, drone) dalam
    satu antarmuka terpadu.
  - Memanfaatkan kecerdasan buatan untuk analisis terrain, rekomendasi
    pola pencarian, dan prediksi pergerakan.
  - Memfasilitasi komunikasi real-time antar tim lapangan dan pusat kendali.

1.2 Pengguna Sistem

  Peran             Deskripsi                    Tingkat Akses
  ----------------  ---------------------------  ----------------------------
  Admin             Administrator sistem         Penuh (seluruh fitur)
  Commander         Koordinator operasi           Dashboard, tracking, AI, chat
  Field Operator    Anggota tim lapangan          Kirim lokasi, chat, lapor
  Viewer            Pemantau/pemangku kepen-      Hanya melihat (baca saja)
                    tingan

2. ARSITEKTUR SISTEM
--------------------------------------------------------------------------------

2.1 Arsitektur Tingkat Tinggi

Sistem dibangun menggunakan arsitektur berlapis (layered architecture)
yang terdiri dari lima lapisan utama:

  Lapisan               Komponen                          Fungsi
  --------------------  --------------------------------  ----------------------
  Lapisan Antarmuka     Dashboard Desktop, Mobile Web     Antarmuka pengguna
  Lapisan API           REST API, WebSocket Server        Routing, autentikasi
  Lapisan Aplikasi      Auth, Tracking, Map, Chat,        Logika bisnis
                        AI/RAG, Stream, Weather
  Lapisan Data          PostgreSQL+PostGIS, Redis,        Penyimpanan data
                        Vector Database
  Layanan Eksternal     API Cuaca, Tile Peta, LLM API,    Integrasi pihak ke-3
                        Aliran CCTV/Drone


2.2 Tumpukan Teknologi

  Lapisan              Teknologi               Justifikasi
  -------------------  ----------------------  ----------------------------
  Frontend             React.js + Vite         Performa tinggi, komponen
  Mesin Peta           Leaflet.js/MapLibre GL  Open-source, multi-layer
  Backend              Node.js + Express       Non-blocking I/O, real-time
  Real-time            Socket.IO               WebSocket + fallback
  Database             PostgreSQL + PostGIS    Kueri geospasial native
  Cache                Redis                   In-memory, pub/sub
  AI/LLM               LangChain + LLM API    Framework RAG fleksibel
  Penyimpanan Vektor   pgvector / ChromaDB     Embedding untuk RAG
  Server Media         Node-Media-Server       Relay RTSP/RTMP ke HLS
  Kontainerisasi       Docker + Compose        Deployment reproducible


2.3 Arsitektur Deployment

Seluruh komponen di-deploy pada server yang disediakan oleh PIHAK KEDUA
menggunakan kontainerisasi Docker. Nginx berfungsi sebagai reverse proxy
dengan SSL termination.

  Layanan                    Kontainer             Port Internal
  -------------------------  --------------------  -------------
  Reverse Proxy + SSL        Nginx                 80, 443
  Frontend (Statis)          Nginx / Node          3000
  Backend API + WebSocket    Node.js               4000
  Database                   PostgreSQL + PostGIS  5432
  Cache                      Redis                 6379
  Server Media               Node-Media-Server     1935, 8000

3. MODUL DAN FITUR SISTEM
--------------------------------------------------------------------------------

3.1 Autentikasi dan Manajemen Pengguna

Sistem autentikasi berbasis JWT (JSON Web Token) dengan kontrol akses
berbasis peran untuk mengatur hak akses setiap pengguna.

  - Mekanisme login/logout dengan JWT access token dan refresh token.
  - Kontrol akses berbasis peran — 4 tingkat akses.
  - Manajemen profil pengguna dan pengelolaan sesi.
  - Pengamanan kata sandi menggunakan bcrypt.


3.2 Dashboard Peta Real-time dan Multi-Layer Map

Dashboard peta interaktif sebagai pusat kendali utama operasi.
Mendukung pergantian layer peta sesuai kebutuhan analisis.

  Layer             Sumber Data                Deskripsi
  ----------------  -------------------------  ----------------------------
  Simple Map        OpenStreetMap              Peta dasar
  Satelit           Mapbox / Esri Satellite    Citra satelit
  Topografi         OpenTopoMap                Kontur elevasi
  Cuaca             OpenWeatherMap             Kondisi cuaca real-time
  Heat Map          Generated internal         Area cakupan pencarian
  Petir             Blitzortung / OWM          Deteksi petir real-time
  Awan              OpenWeatherMap             Tutupan awan
  Prediksi Cuaca    OpenWeatherMap Forecast    Prakiraan 5 hari

Fitur tambahan: Toggle layer, drawing tools, measurement tools,
marker clustering, mode layar penuh, dan tema gelap.


3.3 Pelacakan GPS Real-time

Modul pelacakan posisi seluruh anggota tim lapangan secara real-time
melalui WebSocket, dengan visualisasi trail dan indikator status.

  - Pelacakan posisi langsung dengan interval 5-30 detik (configurable).
  - Visualisasi riwayat rute (trail) pergerakan tim.
  - Indikator kecepatan, arah, dan ketinggian.
  - Peringatan geofencing saat tim keluar zona operasi.
  - Ekspor data pelacakan dalam format GPX/KML.


3.4 Pengiriman Data Lapangan

Mekanisme pengiriman data dan temuan dari lapangan yang otomatis
ditandai pada peta berdasarkan koordinat GPS perangkat pengirim.

  - Formulir pengiriman: lokasi, foto, dan catatan deskriptif.
  - Penandaan otomatis pada peta dengan ikon sesuai kategori.
  - Kategori: Jejak, Barang, Tanda, Titik Bahaya, Titik Istirahat.
  - Antrean offline — data tersimpan lokal, terkirim saat koneksi tersedia.


3.5 Komunikasi Real-time

Sistem percakapan teks real-time berbasis ruang untuk koordinasi antar tim.

  - Arsitektur berbasis ruang (room-based) — kanal per misi/tim.
  - Pesan teks real-time melalui WebSocket.
  - Berbagi lokasi via percakapan (otomatis ditandai pada peta).
  - Berbagi foto dan berkas pendukung.
  - Riwayat pesan dengan fitur pencarian.


3.6 Asisten AI dan RAG

Asisten kecerdasan buatan terintegrasi menggunakan arsitektur
Retrieval-Augmented Generation (RAG) untuk menjawab pertanyaan
berdasarkan basis pengetahuan operasi SAR.

Sumber basis pengetahuan: prosedur standar operasi SAR, data topografi,
panduan survival di lingkungan pegunungan, log operasi sebelumnya,
dan data cuaca historis.

Kapabilitas: antarmuka percakapan, kueri data dengan bahasa natural,
rekomendasi pola pencarian, analisis risiko area, dan ringkasan
status operasi.


3.7 Prediksi Pergerakan Dasar

Modul prediksi berbasis model statistik dan heuristik untuk
memperkirakan kemungkinan lokasi korban.

Parameter input: titik terakhir diketahui (LKP), data terrain,
jalur hiking, sumber air, kondisi cuaca, dan profil korban.

Output: peta probabilitas (heat map), rekomendasi area pencarian,
estimasi radius jangkauan, dan saran pola pencarian.


3.8 Integrasi Aliran Video CCTV

Modul untuk menampilkan aliran video dari kamera CCTV pada posisi
yang sesuai di peta.

  - Penandaan kamera pada peta dengan ikon CCTV.
  - Aliran video langsung dalam popup atau panel samping.
  - Input: aliran RTSP/HLS dari perangkat yang disediakan PIHAK KEDUA.
  - Dukungan minimum 4 aliran simultan.

Prasyarat: PIHAK KEDUA menyediakan kamera CCTV yang telah terkoneksi
jaringan dan dapat diakses melalui protokol RTSP atau HLS.


3.9 Integrasi Aliran Video Drone

Modul untuk menampilkan aliran video langsung dari drone pada peta,
termasuk pelacakan posisi drone jika data telemetri tersedia.

  - Tampilan aliran video drone langsung pada peta.
  - Input: aliran RTMP/WebRTC dari drone yang disediakan PIHAK KEDUA.
  - Integrasi berupa stream relay — bukan integrasi SDK vendor spesifik.

Prasyarat: PIHAK KEDUA menyediakan drone dengan kemampuan streaming
langsung (output RTMP/WebRTC).

4. MODEL DATA
--------------------------------------------------------------------------------

  Entitas          Atribut Utama                            Relasi
  ---------------  ---------------------------------------  --------------------
  Users            id, username, email, password_hash,      Memiliki: Tracking,
                   role, is_active                          Reports, Messages

  Missions         id, name, description,                   Memiliki: Tracking,
                   last_known_position, status               Reports, ChatRooms

  TrackingData     id, user_id, mission_id, position,       Milik: Users,
                   altitude, speed, heading                  Missions

  FieldReports     id, user_id, mission_id, location,       Milik: Users,
                   category, description, attachments        Missions

  ChatRooms        id, mission_id, name, type               Milik: Missions
                                                             Memiliki: Messages

  ChatMessages     id, room_id, user_id, content,           Milik: ChatRooms,
                   type, metadata                            Users

  StreamSources    id, name, type, stream_url,              Mandiri
                   location, status

Database menggunakan PostgreSQL dengan ekstensi PostGIS untuk tipe data
geospasial dan kueri spasial.

6. PERSYARATAN NON-FUNGSIONAL
--------------------------------------------------------------------------------

  Persyaratan                  Target
  ---------------------------  ---------------------------------
  Waktu respons API            < 200 milidetik (P95)
  Waktu muat peta              < 3 detik
  Latensi pembaruan GPS        < 2 detik end-to-end
  Pengguna simultan            Minimum 50 pengguna
  Aliran video simultan        Minimum 4 aliran
  Ketersediaan sistem          99% (bergantung infrastruktur)
  Dukungan peramban            Chrome, Firefox, Safari, Edge
  Dukungan mobile              Responsif, min viewport 360px


--------------------------------------------------------------------------------
7. ASUMSI DAN BATASAN
--------------------------------------------------------------------------------

7.1 Asumsi
  1. Server memiliki spesifikasi minimum: 8 CPU cores, 16 GB RAM, 100 GB SSD.
  2. Server memiliki akses internet stabil untuk layanan API eksternal.
  3. Perangkat CCTV dan drone telah memiliki kemampuan streaming.
  4. Tim lapangan menggunakan perangkat dengan peramban modern dan GPS.
  5. Koneksi internet di area operasi tersedia minimal 3G/4G.

7.2 Batasan
  1. Platform berupa aplikasi web responsif — bukan mobile native.
  2. Prediksi pergerakan menggunakan model statistik — bukan deep learning.
  3. Integrasi drone terbatas pada stream relay — bukan SDK vendor.
  4. Fitur offline terbatas pada antrean pengiriman data.
