1. Ringkasan Eksekutif

Project Tracking & Intelligence Platform adalah platform perangkat lunak berbasis web yang dirancang untuk mendukung operasi pelacakan, pencarian, komunikasi, monitoring, dan analisis data pada wilayah atau area tertentu. Platform ini mengintegrasikan teknologi pelacakan geolokasi real-time, auto alert, analitik data, peta interaktif multi-layer, analisis berbasis kecerdasan buatan, dan komunikasi terpadu dalam satu antarmuka.

1.1 Tujuan Sistem

  • Menyediakan Common Operating Picture (COP) real-time untuk koordinasi operasi pelacakan, pencarian, komunikasi, monitoring, dan analisis data pada wilayah atau area tertentu.
  • Melacak posisi seluruh anggota tim lapangan secara real-time pada peta interaktif multi-layer.
  • Mengintegrasikan data dari berbagai sumber — GPS, CCTV, dan drone — dalam satu antarmuka terpadu.
  • Memanfaatkan kecerdasan buatan untuk analisis terrain, rekomendasi pola pencarian, dan prediksi pergerakan.
  • Memfasilitasi komunikasi real-time antar tim lapangan dan pusat kendali.

3. Modul dan Fitur Sistem

3.1 Autentikasi dan Manajemen Pengguna

Sistem autentikasi berbasis JWT (JSON Web Token) dengan kontrol akses berbasis peran untuk mengatur hak akses setiap pengguna sesuai perannya dalam operasi.

  • Mekanisme login/logout dengan JWT access token dan refresh token.
  • Kontrol akses berbasis peran (Role-Based Access Control) — 4 tingkat akses.
  • Manajemen profil pengguna dan pengelolaan sesi aktif.
  • Pengamanan kata sandi menggunakan algoritma Argon2.

3.3 Pelacakan GPS Real-time

Modul pelacakan posisi seluruh anggota tim lapangan secara real-time melalui WebSocket, dengan visualisasi trail dan indikator status pada peta.

Tampilan mobile untuk operator lapangan:

  • Pelacakan posisi langsung pada peta dengan interval pembaruan 5–30 detik (dapat dikonfigurasi).
  • Visualisasi riwayat rute (trail) pergerakan tim.
  • Indikator kecepatan, arah, dan ketinggian.
  • Peringatan geofencing saat tim keluar dari zona operasi.
  • Ekspor data pelacakan dalam format GPX/KML.

3.4 Pengiriman Data Lapangan

Mekanisme pengiriman data dan temuan dari lapangan yang otomatis ditandai pada peta berdasarkan koordinat GPS perangkat pengirim atau berdasarkan input operator pusat operasi.

Formulir Pengiriman Laporan (Foto, Video, Suara, Dokumen):

  • Formulir pengiriman: lokasi, foto, video, suara, dan catatan deskriptif.
  • Penandaan otomatis pada peta dengan ikon sesuai kategori.
  • Kategori temuan: Jejak, Barang, Tanda, Titik Bahaya, Titik Istirahat.
  • Integrasi saluran komunikasi radio atau alat komunikasi lain yang digunakan dalam operasi. (Catatan: bergantung pada ketersediaan perangkat dan protokol yang didukung.)
  • Antrean offline — data tersimpan secara lokal dan terkirim otomatis saat koneksi tersedia.

3.5 Komunikasi Real-time

Sistem percakapan teks real-time berbasis ruang untuk koordinasi antar tim.

Detail Komunikasi (Teks, Foto, Video, Suara, Lokasi, Dokumen, Radio HT):

  • Arsitektur berbasis ruang (room-based) — kanal per misi atau per tim.
  • Pesan teks real-time melalui WebSocket.
  • Berbagi lokasi via percakapan (otomatis ditandai pada peta).
  • Berbagi foto dan berkas pendukung.
  • Riwayat pesan dengan fitur pencarian.

3.6 Asisten AI dan RAG

Asisten kecerdasan buatan terintegrasi yang menggunakan arsitektur Retrieval-Augmented Generation (RAG) untuk menjawab pertanyaan berdasarkan basis pengetahuan operasi SAR dan bentuk operasi lainnya.

Sumber basis pengetahuan: prosedur standar operasi, data topografi, panduan survival, log operasi sebelumnya, dokumen dan laporan yang diunggah, serta data cuaca historis.

Kapabilitas: antarmuka percakapan, kueri data dengan bahasa natural, rekomendasi pola pencarian, analisis risiko area, ringkasan status operasi, dan estimasi lokasi.

3.7 Prediksi Pergerakan Dasar

Modul prediksi berbasis model statistik dan heuristik untuk memperkirakan kemungkinan lokasi korban.

Parameter input: titik terakhir diketahui (LKP), data terrain, jalur hiking, sumber air, kondisi cuaca, dan profil korban.

Output: peta probabilitas (heat map), rekomendasi area pencarian, estimasi radius jangkauan, dan saran pola pencarian.

3.8 Integrasi Aliran Video CCTV

Modul untuk menampilkan aliran video dari kamera CCTV pada posisi yang sesuai di peta.

Flow Integrasi CCTV dan Drone (Protokol dan Arsitektur):

  • Penandaan kamera pada peta dengan ikon CCTV.
  • Aliran video langsung dalam popup atau panel samping.
  • Input: aliran RTSP/HLS dari perangkat yang disediakan PIHAK KEDUA.
  • Dukungan minimum 4 aliran simultan.
Prasyarat

PIHAK KEDUA menyediakan kamera atau link streaming CCTV yang telah terkoneksi jaringan dan dapat diakses melalui protokol RTSP, HLS, atau format lainnya.

3.9 Integrasi Aliran Video Drone

Modul untuk menampilkan aliran video langsung dari drone pada peta, termasuk pelacakan posisi drone jika data telemetri tersedia.

  • Tampilan aliran video drone langsung pada peta.
  • Input: aliran RTMP/WebRTC dari drone yang disediakan PIHAK KEDUA.
  • Kontrol CCTV dan drone jarak jauh.
  • Integrasi berupa stream relay — bukan integrasi SDK vendor spesifik.
Prasyarat

PIHAK KEDUA menyediakan drone dengan kemampuan streaming langsung (output RTMP/WebRTC).

7. Asumsi dan Batasan

7.1 Asumsi

  1. Server yang disediakan memiliki spesifikasi minimum:
    • 8 CPU cores, 32 GB RAM, 100 GB SSD (untuk pengembangan backend)
    • 8 CPU cores, 8 GB RAM, 100 GB SSD (untuk pengembangan frontend)
    • 8 CPU cores, 8 GB RAM, 300 GB SSD (untuk database khusus)
    • 8 CPU cores, 8 GB RAM, 300 GB SSD (untuk monitoring tools Grafana)
  2. Server memiliki akses internet stabil untuk koneksi ke layanan API eksternal.
  3. Perangkat CCTV dan drone yang disediakan telah memiliki kemampuan streaming (RTSP/RTMP/HLS).
  4. Tim lapangan menggunakan perangkat dengan peramban modern dan modul GPS aktif.
  5. Koneksi internet di area operasi tersedia minimal pada level 2G/3G/4G.
  6. PIHAK KEDUA menyediakan seluruh API key dan kredensial layanan berbayar sebelum fase terkait dimulai.

7.2 Batasan

  1. Platform berupa aplikasi web responsif — bukan aplikasi mobile native.
  2. Prediksi pergerakan menggunakan model statistik/heuristik — bukan deep learning.
  3. Integrasi drone terbatas pada stream relay — bukan integrasi SDK vendor.
  4. Fitur offline terbatas pada antrean pengiriman data.
  5. Kendali drone jarak jauh bersifat experimental — terbatas pada kontrol kamera jika didukung perangkat.
  6. Integrasi radio HT bergantung pada ketersediaan perangkat dan protokol yang didukung.
