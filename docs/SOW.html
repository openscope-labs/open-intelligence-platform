1. Pendahuluan

PIHAK KEDUA membutuhkan pengembangan platform perangkat lunak berbasis web untuk mendukung operasi pelacakan, pencarian, komunikasi, monitoring, dan analisis data pada wilayah atau area tertentu. Platform ini harus mampu menampilkan posisi tim lapangan secara real-time, mengintegrasikan data dari berbagai sumber, serta menyediakan kemampuan analisis berbasis kecerdasan buatan.

Dokumen Statement of Work ini mendefinisikan secara mengikat ruang lingkup pekerjaan, deliverables, timeline, dan ketentuan terkait lainnya. Segala pekerjaan yang tidak tercantum dalam dokumen ini dianggap berada di luar ruang lingkup dan akan dikenakan biaya tambahan melalui prosedur Perubahan Ruang Lingkup sebagaimana diatur pada Bagian 9.

3. Ruang Lingkup Pekerjaan

A. Dashboard Peta Real-time dan Multi-Layer Map

  • Dashboard peta interaktif sebagai pusat kendali operasi.
  • 10 (sepuluh) layer peta dasar: Simple Map, Satelit, Topografi, Cuaca, Heat Map, Petir, Awan, Prediksi Cuaca, Rute & Posisi Penerbangan Pesawat (OpenSky / FlightRadar24), serta Jalur Kapal Laut (AISStream / MarineTraffic).
  • Perangkat menggambar (drawing tools) untuk penandaan area: marker, polyline, polygon, circle, dilengkapi karva terkustomisasi.
  • Mode layar penuh dan tema gelap untuk pusat kendali.

B. Pelacakan GPS Real-time

  • Pelacakan posisi seluruh anggota tim lapangan secara real-time.
  • Visualisasi riwayat rute (trail), indikator kecepatan dan arah.
  • Interval pembaruan yang dapat dikonfigurasi (5–30 detik).
  • Peringatan geofencing dan ekspor data dalam format GPX/KML.

C. Pengiriman Data dari Lapangan

  • Formulir pengiriman data berupa lokasi, foto, dan catatan dari perangkat mobile.
  • Penandaan otomatis pada peta berdasarkan koordinat GPS perangkat.
  • Kategorisasi temuan: jejak, barang, tanda, titik bahaya, dan titik istirahat.
  • Antrean offline (offline queue) dengan sinkronisasi otomatis saat koneksi tersedia.

D. Komunikasi Real-time

  • Sistem percakapan berbasis ruang (room-based) per misi atau per tim.
  • Pesan teks real-time dengan dukungan berbagi lokasi, foto, video, dan suara.
  • Integrasi komunikasi berbasis radio HT atau alat komunikasi sinyal lainnya. (Catatan: bersifat eksperimental — bergantung pada ketersediaan perangkat dan protokol yang didukung.)
  • Riwayat pesan dengan fitur pencarian.

E. Asisten AI dan RAG

  • Antarmuka percakapan dengan asisten kecerdasan buatan berbasis RAG.
  • Basis pengetahuan: prosedur SAR, data terrain, dan panduan survival, segala dokumen, pesan dan informasi yang dilaporkan, dan data cuaca historis.
  • Kemampuan query data operasi dengan bahasa natural dan rekomendasi pola pencarian.
  • Rekomendasi area potensial untuk rekayasa cuaca / hujan buatan berdasarkan agregasi data dari API pihak ketiga.

F. Visualisasi Analitik, Probabilitas, dan Prediksi

  • Visualisasi Heatmap Data Telekomunikasi Geospasial (Parsing data MSISDN/CID). Model dan pemrosesan heatmap sepenuhnya disediakan oleh PIHAK KEDUA.
  • Visualisasi Prediksi Pergerakan Entitas (misal: orang hilang). Model prediksi pergerakan (arah, estimasi radius) sepenuhnya disediakan oleh PIHAK KEDUA.
  • Visualisasi proyeksi dan probabilitas posisi entitas pada peta (Model disediakan oleh PIHAK KEDUA).
  • Tim PIHAK PERTAMA hanya bertanggung jawab membangun mekanisme consumer (API/WebSocket) dan memvisualisasikannya di antarmuka peta.

G. Integrasi Aliran Video CCTV

  • Tampilan aliran video CCTV langsung pada peta dalam bentuk popup atau panel samping.
  • Dukungan berbagai protokol (RTSP, HLS, RTMP, Web Embed) dari perangkat kamera milik PIHAK KEDUA atau pihak ketiga.
  • Integrasi Deteksi Keramaian dari aliran CCTV. (Model/Endpoint Computer Vision disediakan oleh PIHAK KEDUA).

H. Integrasi Aliran Video Drone

  • Tampilan aliran video drone langsung dan pelacakan posisi drone pada peta.
  • Input: aliran RTMP/WebRTC dari drone milik PIHAK KEDUA.
  • Proyeksi kamera drone terhadap peta secara real-time.
  • Kendali drone jarak jauh. (Catatan: cakupan kendali terbatas pada kontrol kamera — kendali navigasi penerbangan bersifat eksperimental dan bergantung pada kompatibilitas perangkat.)
  • Integrasi berupa stream relay — bukan integrasi SDK vendor spesifik. (Catatan: driver kustom akan dikembangkan sebagai bagian dari riset awal integrasi.)

I. Aplikasi Web Responsif

  • Tampilan desktop: dashboard pusat kendali.
  • Tampilan mobile: antarmuka untuk tim lapangan.
  • Kompatibel dengan Chrome, Firefox, Safari, dan Edge.

J. Manajemen Pengguna

  • Autentikasi berbasis JWT dengan 4 tingkat akses (Admin, Commander, Field Operator, Viewer).
  • Kontrol akses berbasis peran (role-based access control).

K. Deployment Produksi

  • Deployment ke server PIHAK KEDUA menggunakan Docker dan Kubernetes.
  • Konfigurasi Nginx reverse proxy dan SSL/HTTPS (Let's Encrypt).
  • Pengaturan monitoring (Grafana + Prometheus) dan logging dasar.

8. Tanggung Jawab Para Pihak

8.1 PIHAK PERTAMA (Openscopelabs)

  1. Merancang dan mengembangkan seluruh fitur sesuai ruang lingkup SOW.
  2. Menyerahkan deliverables sesuai timeline.
  3. Melaksanakan demonstrasi di setiap pencapaian milestone.
  4. Memperbaiki cacat perangkat lunak selama masa pengembangan dan garansi.
  5. Menyerahkan kode sumber dan dokumentasi teknis.
  6. Melakukan deployment ke server PIHAK KEDUA ketika pembayaran termin telah terpenuhi.

8.2 PIHAK KEDUA (Cakra Inteligence)

  1. Melakukan pembayaran sesuai jadwal yang disepakati.
  2. Menyediakan server untuk pengembangan dan produksi.
  3. Menyediakan akses ke perangkat keras (CCTV, drone) untuk keperluan integrasi.
  4. Menyediakan endpoint API atau Model AI yang telah berjalan untuk fitur: Deteksi Keramaian CCTV, Telco Heatmap Geospasial, Prediksi Pergerakan, dan Probabilitas Posisi.
  5. Menyediakan dan menanggung biaya layanan API pihak ketiga jika ada batasan (kuota) dari layanan gratis, atau jika PIHAK KEDUA memutuskan untuk menggunakan API Komersial Berbayar (misal: FlightRadar24 Enterprise API atau MarineTraffic API).
  6. Menunjuk 1 (satu) penanggung jawab (Person in Charge).
  7. Memberikan umpan balik atau persetujuan dalam 5 hari kerja setelah demonstrasi.
  8. Menyediakan materi dasar untuk basis pengetahuan AI.

9. Prosedur Perubahan Ruang Lingkup

Setiap permintaan perubahan atau penambahan fitur di luar SOW ini wajib melalui prosedur berikut:

  1. PIHAK KEDUA mengajukan permintaan perubahan secara tertulis.
  2. PIHAK PERTAMA melakukan analisis dampak dalam 3 (tiga) hari kerja.
  3. PIHAK PERTAMA menyampaikan estimasi biaya tambahan dan dampak terhadap timeline.
  4. Biaya tambahan dihitung berdasarkan tarif Rp 500.000,-/orang x hari kerja.
  5. Pekerjaan tambahan dimulai setelah addendum SOW ditandatangani oleh Para Pihak.

10. Kriteria Penerimaan

Fase 1 — Platform Inti

  • Pengguna dapat masuk sesuai peran yang ditetapkan.
  • Peta interaktif menampilkan minimal 3 layer.
  • Pelacakan GPS menampilkan posisi real-time minimal 2 perangkat.
  • Pengiriman data lapangan berhasil ditandai pada peta.

Fase 2 — Komunikasi

  • Percakapan real-time berfungsi antar 2 pengguna atau lebih.
  • Layer cuaca, awan, dan petir tampil pada peta.
  • Heat map menampilkan area cakupan pencarian.

Fase 3 — Intelijen

  • Asisten AI merespons pertanyaan terkait operasi pencarian.
  • Peta prediksi muncul berdasarkan masukan titik terakhir diketahui.

Fase 4 — Integrasi dan Deployment

  • Aliran video CCTV dan drone tampil pada peta.
  • Sistem berjalan di server produksi PIHAK KEDUA.
  • Seluruh fitur berfungsi di peramban desktop dan mobile.

Pengujian Penerimaan: Dilaksanakan selama 5 hari kerja setelah serah terima. Cacat yang termasuk dalam ruang lingkup diperbaiki dalam 10 hari kerja. Setelah penerimaan ditandatangani, proyek dinyatakan selesai.
